# 2026-07-03 Progress Summary

## Main Pipeline Progress

Tonight's main line moved the project toward a portable image-to-isosceles-triangles workflow:

1. CUDA greedy prior placement remains the structural initializer.
2. Attention weighting now guides greedy placement.
3. Dynamic-resolution greedy placement was added:
   - first 20% of triangles at 256 px without attention
   - 20%-50% at 512 px without attention
   - final 50% at 1024 px with attention
4. Constrained diffvg refinement keeps isosceles triangle parameters and fixed alpha.
5. Rebirth now uses attention for deletion, residual sampling, and reborn greedy search, while diffvg's main optimization loss remains ordinary L1.

## Attention Work

MediaPipe-based attention mask generation was added under `trianglefit.attention`.

Typical mask generation command:

```powershell
.\.venv\Scripts\python.exe -m trianglefit.attention.mediapipe_mask `
  --input assets\zhongli_030.png `
  --output out\zhongli_attention_mask `
  --work-size 512
```

Generated outputs include:

- `attention_mask.pt`
- `attention_mask.npy`
- `attention_mask.png`
- `attention_overlay.png`
- `attention_metadata.json`

Observed behavior:

- `zhongli_030.png` works with MediaPipe and produces eyes/nose/mouth attention.
- `linaiya.png` was not detected by MediaPipe in current tests, so it needs either a better anime face detector or manual/fallback regions later.

## Greedy Improvements

Greedy scoring now supports optional attention:

```text
score = sum(attention * (target - rendered)^2)
color = sum(attention * target) / sum(attention)
```

Dynamic-resolution greedy config:

```powershell
.\.venv\Scripts\python.exe -m trianglefit.greedy_prior.place `
  --config configs\greedy_dynamic_resolution_attention_cuda.json
```

The dynamic scheduler rescales already placed triangles between stages and rerenders them as the next stage's initial image.

## Diffvg And Rebirth

The current preferred refinement command from the dynamic greedy result is:

```powershell
.\.venv\Scripts\python.exe -m trianglefit.direct.fit_diffvg_isosceles_backend `
  --input assets\zhongli_030.png `
  --init-json out\zhongli_greedy_dynamic_attention\greedy_geometrize.json `
  --output out\zhongli_greedy_dynamic_attention_diffvg `
  --work-size 512 `
  --device cuda `
  --steps 300 `
  --samples 2 `
  --geometry-lr 0.02 `
  --color-lr 0.01 `
  --rebirth-count 10 `
  --rebirth-every 100 `
  --rebirth-initial `
  --rebirth-candidate-count 64 `
  --rebirth-max-shape-mutations 512 `
  --rebirth-bounds-fraction 0.2 `
  --rebirth-attention-mask out\zhongli_attention_mask\attention_mask.pt
```

Important distinction:

- Greedy and rebirth use attention.
- diffvg's main optimization loss intentionally remains ordinary L1.
- Alpha remains fixed/opaque and is not optimized.

## Experimental Branch

`trianglefit/direct/fit_diffvg_backend.py` contains an experimental non-mainline diffvg backend change:

- Gaussian blur coarse-to-fine annealing.
- Optional LPIPS/VGG-style perceptual loss.

This is not the current isosceles constrained main path. It is kept for future experiments.
