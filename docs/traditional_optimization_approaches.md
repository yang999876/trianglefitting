# 传统优化路线探索：突破 Geometrize + diffvg 的局部最优

## 背景

当前主线工作流：Geometrize 贪心初始化 → pydiffvg 梯度下降优化。

核心瓶颈：diffvg 的梯度下降只能微调颜色，几何结构跳不出 Geometrize 给出的布局。根本原因：

1. **遮挡顺序是离散的**：三角形 A 盖在 B 上 vs B 盖在 A 上，loss landscape 完全不同，梯度无法跨越这种拓扑变化
2. **三角形数量分配不合理**：Geometrize 贪心倾向于用大三角形铺底，某些区域冗余堆积，细节区域预算不足
3. **梯度消失/冲突**：被完全遮挡的三角形收不到梯度；重叠区域的梯度信号互相干扰
4. **贪心分工锁定**：Geometrize 每步决定一个三角形负责哪块区域，一旦分工不合理，后续梯度微调无法修正

以下思路均不涉及神经网络，聚焦于组合搜索、调度策略和优化流程的改进。

---

## 思路 1：分层渐进式重优化（Coarse-to-Fine Reoptimization）

不一次性优化所有三角形，按层级分阶段解锁。

### 流程

- **Phase 1**：只优化前 N_base 个大三角形（铺底色块），冻结其余
- **Phase 2**：锁定底层，优化中层三角形——不是微调，而是**允许大幅移动**（高学习率或重新初始化位置到当前残差最大区域）
- **Phase 3**：解锁所有层做联合微调

### 核心差异

Phase 2 的关键不是微调，而是**用当前残差图引导中层三角形重新分配**。这绕过了 Geometrize 贪心序列决策中分配不合理的问题。

### 与现有代码的关系

仓库里的 `FitConfig` 已有 base/texture/accent 三层角色系统和 growth schedule，但目前是在自研渲染器里用的。可以将类似的分层调度迁移到 diffvg 后端。

---

## 思路 2：周期性 Shakeup（冻结好的 + 重新搜索差的）

**融合 Geometrize 的随机搜索能力和 diffvg 的梯度精调。**

### 流程

```
loop:
  1. 用 diffvg 梯度下降优化所有三角形（几百步）
  2. 评估每个三角形的贡献度（移除它后 loss 增加多少）
  3. 保留贡献高的 top-K，删除剩余的
  4. 对残差图，用类似 Geometrize 的随机搜索补充新三角形
  5. 回到 1
```

### 核心思想

交替优化：梯度下降负责连续参数精调，随机搜索负责离散结构探索（添加/替换/重分配）。

### 贡献度评估方法

- **精确法**：逐个移除三角形重渲染，计算 delta_loss（开销 O(N) 次渲染）
- **近似法**：用三角形的 mask 面积 × 该区域残差均值估算贡献（开销极低）
- **梯度法**：看参数梯度的范数，梯度接近零的三角形可能已经"死了"或完全被遮挡

### 与现有代码的关系

仓库里的 rebirth 机制（`_rebirth_role`、`_lowest_contribution_indices`）已有贡献度评估和重初始化逻辑，但只在自研软渲染器中运行。核心改动：在 diffvg 优化循环中接入同样的"评估 → 删除 → 随机搜索补充"。

## 思路 3：渲染顺序优化（Permutation Search）

遮挡顺序对最终图像影响巨大，但梯度对排列完全无能为力。这是当前流程中被完全忽略的一个优化维度。

### 方法 A：贪心重排

每隔 N 步，对所有三角形做一次贪心排列优化：
- 逐个把三角形从当前位置取出
- 尝试插入到不同深度位置
- 保留使 loss 最小的位置

复杂度：每个三角形尝试 O(N) 个位置，总共 O(N²) 次渲染（可以用增量渲染加速）。

### 方法 B：相邻交换 + 模拟退火

- 每步随机选两个相邻三角形交换顺序
- 用 Metropolis 准则决定是否接受（温度逐步退火）
- 计算开销小——交换相邻两个只需局部重渲染

### 方法 C：启发式排序

按规则自动排序作为更好的初始排列：
- 按面积降序（大的在下、小的在上）
- 按亮度排序（暗色在下、亮色在上）
- 按与背景色的差异排序

### 与梯度下降的结合

渲染顺序优化和参数梯度下降是正交的——可以交替执行：
```
for round in range(R):
    梯度下降优化参数 (M 步)
    贪心/退火优化渲染顺序
```

## 思路 4：超参数化 + 剪枝（Over-parameterize then Prune）

受神经网络剪枝的启发。

### 流程

1. 用 **2-3 倍于目标数量** 的三角形开始优化
2. 每个三角形带一个可学习的 importance 权重（可以直接用 alpha 通道）
3. 优化结束后按 importance 排序，剪掉分数最低的
4. 用剩余三角形做 fine-tune

### 在 diffvg 中的实现

diffvg 的三角形已经有 RGBA 四通道，alpha 本身就是可学习的。让 alpha 参与梯度下降：
- 不重要的三角形 alpha 会自然趋近 0
- 优化后删除 alpha < threshold 的三角形
- 用节省下来的预算在高残差区域补充新三角形

### 优势

让梯度下降自己决定哪些三角形该存在、在哪里、多大——比 Geometrize 的贪心序列决策更"全局"。

### 需要注意

- 2-3 倍的三角形会显著增加渲染和反传开销
- 可能需要加 L1 稀疏正则化来鼓励 alpha 稀疏化

## 思路 5：多起点 + 精英选择（Multi-Start with Elite Seeding）

利用 Geometrize 的随机性，通过多次运行和交叉组合来扩大搜索范围。

### 流程

```
1. 跑 K 次不同 seed 的 Geometrize（K=5-10）
2. 每次结果用 diffvg 短暂优化（200 步）
3. 从 K 个结果中选最好的 1-2 个作为精英
4. 交叉：从不同精英中取子集三角形组合出新个体
5. 继续下一轮
```

### 交叉策略

两个不同的 Geometrize 结果可能在不同区域做得好。交叉方法：
- **空间分区交叉**：将画面分成若干区域，每个区域从表现最好的那个精英中取三角形
- **贡献度交叉**：从每个精英中各取贡献度最高的 top-50%，合并去重

### 与现有代码的关系

仓库里已有 `run_diffvg_multistart.py`，目前是多起点并行。扩展方向：加入精英选择和交叉组合。

## 思路 6：区域分治（Spatial Decomposition）

将图像分成 NxN 的网格区域，每个区域独立优化自己的三角形子集。

### 流程

1. 将图像划分为 NxN 网格（如 4x4 = 16 个区域）
2. 按残差大小给各区域分配不同数量的三角形预算
3. 各区域独立跑 Geometrize + diffvg 优化
4. 拼合所有区域的三角形
5. 全局微调一轮，处理接缝

### 优势

- 每个子问题规模小，搜索空间指数级缩小，更容易找到好的局部解
- 天然可并行

### 挑战

- 跨区域的三角形需要特殊处理（大三角形可能横跨多个区域）
- 接缝处可能出现不连续

### 改进

- 两级分治：先粗粒度区域放大三角形，再细粒度区域放小三角形
- 允许三角形在全局微调阶段自由越界

## 思路 7：残差级联（Residual Cascading）

受 Boosting / 残差学习的启发，逐轮拟合残差而非一次性拟合全图。

### 流程

```
round 1: 用 100 个三角形拟合原图，得到 image_1
round 2: 残差 residual_1 = target - image_1
         放大残差对比度，用 100 个新三角形拟合
round 3: residual_2 = target - image_1 - image_2
         再用 100 个三角形拟合
...
最终: 把所有三角形放在一起，用 diffvg 全局微调
```

### 优势

- 每一轮只需要关注"当前错在哪里"，避免了大量三角形同时优化时的梯度干扰
- 自然形成从粗到细的层次结构：早期轮次拟合大色块，后期轮次拟合细节
- 每轮的子问题规模小，Geometrize 和 diffvg 都更容易找到好解

### 实现要点

- 残差可能有负值，需要做偏移和缩放后再拟合（或用两组三角形分别拟合正/负残差）
- 最后的全局微调步骤至关重要——让所有轮次的三角形重新协调

---

## 优先级建议

| 优先级 | 方向 | 理由 |
|:---:|------|------|
| **1** | 思路 2：周期性 Shakeup | 直接复用现有 rebirth 逻辑 + diffvg，实现成本最低，效果预期最好 |
| **2** | 思路 3：渲染顺序优化 | 实现简单，当前完全被忽略的优化维度，独立于其他方法 |
| **3** | 思路 7：残差级联 | 思路清晰、实现独立，可与现有 Geometrize 工具链无缝结合 |
| **4** | 思路 1：分层渐进式重优化 | 现有 growth schedule 可以适配，但需要较多调参 |
| **5** | 思路 4：超参数化 + 剪枝 | 在 diffvg 中放开 alpha 学习即可，但渲染开销翻倍需要权衡 |
| **6** | 思路 5：多起点 + 精英交叉 | multistart 已有，加交叉不难，但整体提升可能有限 |
| **7** | 思路 6：区域分治 | 工程量较大，接缝问题需要额外处理 |

思路 2 和思路 3 可以同时实现、叠加使用——Shakeup 负责"在哪里放三角形"，顺序优化负责"谁盖谁"。
